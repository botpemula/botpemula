    private $cookie;

    {
        $this->useragent = 'Mozilla/5.0 (X11; Ubuntu; Linux i686; rv:40.0)';
        $this->cookie = '';
    }

    private function requestHeaders($method = 'GET')
    {
        $headers = [
            'user-agent: ' . $this->useragent,
            'cookie: ' . $this->cookie,
        ];
                      
        if ($method == 'POST')
        {
            $headers[] = 'content-type: application/x-www-form-urlencoded';
        }
                      
        return $headers;

Menbunyikan Header

Untuk menghindari deteksi multi-akun dan penggunaan proxy dalam skrip PHP Anda, terdapat beberapa langkah yang bisa Anda coba. Namun, perlu diingat bahwa ini bukanlah solusi yang 100% pasti menghindari deteksi, karena banyak situs yang memiliki sistem pendeteksian yang sangat canggih dan dapat mendeteksi pola-pola tertentu.Berikut adalah beberapa langkah yang dapat Anda coba untuk menyembunyikan penggunaan proxy dan multi-akun:

1. Menggunakan Proxy Rotator (Rotasi Proxy). Jika Anda menggunakan proxy, pastikan Anda menggunakan proxy yang berganti secara otomatis untuk setiap permintaan. Ini dapat mengurangi kemungkinan deteksi jika hanya satu IP yang digunakan terus-menerus.Gunakan layanan rotasi proxy (misalnya, ProxyMesh atau Smartproxy). Menggunakan proxy dengan IP yang berasal dari Indonesia, atau negara yang sesuai dengan situs yang Anda tuju. Contoh dibawah ini penggunaan proxy yang lebih fleksibel dalam cURL:

																		
<?php

$proxyList = [
    'proxy1:port', 
    'proxy2:port', 
    'proxy3:port',
];

$randomProxy = $proxyList[array_rand($proxyList)]; 

$curl = curl_init();
curl_setopt($curl, CURLOPT_URL, $url);
curl_setopt($curl, CURLOPT_RETURNTRANSFER, true);
curl_setopt($curl, CURLOPT_PROXY, $randomProxy);
curl_setopt($curl, CURLOPT_PROXYTYPE, CURLPROXY_HTTP);

?>

2. Mengganti User-Agent dan Headers. Bergantung pada situs yang Anda tuju, penggunaan user-agent yang berbeda bisa membantu menghindari deteksi. Anda bisa merotasi user-agent dan header lainnya untuk meniru perilaku pengguna biasa. Contoh rotasi user-agent:

																		
<?php

$userAgents = [
    'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/58.0.3029.110 Safari/537.36',
    'Mozilla/5.0 (Windows NT 6.1; WOW64; rv:40.0) Gecko/20100101 Firefox/40.0',
    'Mozilla/5.0 (Windows NT 6.3; rv:36.0) Gecko/20100101 Firefox/36.0'
];

$randomUserAgent = $userAgents[array_rand($userAgents)];  // Pilih User-Agent secara acak

$headers = [
    'User-Agent: ' . $randomUserAgent,
    'Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8,application/signed-exchange;v=b3;q=0.7',
    'Accept-Language: id-ID,id;q=0.9,en-US;q=0.8,en;q=0.7',
];

$curl = curl_init();
curl_setopt($curl, CURLOPT_HTTPHEADER, $headers);

?>

3. Menggunakan VPN dengan IP Lokal. Jika situs mendeteksi penggunaan proxy, Anda mungkin perlu mempertimbangkan untuk menggunakan VPN yang memiliki server di Indonesia. Dengan menggunakan VPN, Anda akan memiliki IP lokal yang lebih mirip dengan pengguna asli.

4. Menyembunyikan Jejak Cookie dan Cache.Saat menggunakan proxy atau VPN, pastikan Anda juga memperhatikan cookie dan cache yang digunakan pada permintaan. untuk menyimpan dan memuat cookie agar situs tidak mendeteksi aktivitas yang mencurigakan.Gunakan metode seperti:

																			
<?php

$curl = curl_init();
curl_setopt($curl, CURLOPT_COOKIEJAR, 'cookies.txt');
curl_setopt($curl, CURLOPT_COOKIEFILE, 'cookies.txt');

?>

5. Menghindari Penggunaan IP yang Teridentifikasi. Jika IP Anda sudah terdeteksi sebelumnya, jangan gunakan IP yang sama. Rotasi IP secara teratur jika memungkinkan.

6. Menggunakan Captcha Solver (Jika diperlukan). Beberapa situs mungkin menggunakan Captcha untuk mencegah bot. Jika Anda menggunakan bot untuk mengklaim atau mengakses situs secara otomatis, Anda mungkin perlu menggunakan layanan captcha solver (misalnya 2Captcha) untuk menyelesaikan captcha secara otomatis.

7. Jangan Terlalu Cepat dalam Mengirimkan Permintaan.Mengirimkan permintaan terlalu cepat dapat membuat sistem mendeteksi bahwa ini adalah aktivitas bot. Pastikan Anda memberikan jeda waktu yang cukup di antara setiap permintaan, misalnya dengan menggunakan sleep() dalam skrip.Menambahkan delay agar tidak terlihat seperti bot. Delay antara 3 sampai 7 detik.

																					
<?php

sleep(rand(3, 7));

?>

8. Penyembunyian Jejak Header. Selain mengubah User-Agent, Anda juga dapat memodifikasi header lainnya untuk meniru perilaku pengguna manusia misalnya seperti ini.

																					
<?php

$headers = [
    'User-Agent: ' . $randomUserAgent,
    'Accept: text/html,application/xhtml+xml,application/xml;q=0.9,image/avif,image/webp,image/apng,*/*;q=0.8',
    'Accept-Language: id-ID,id;q=0.9,en-US;q=0.8,en;q=0.7',
    'Connection: keep-alive',
    'Upgrade-Insecure-Requests: 1',
    'Cache-Control: max-age=0',
];

?>

9. Analisa Waktu Aktivitas. Aktivitas Anda di situs harus terlihat alami. Misalnya, jangan terlalu sering melakukan permintaan atau klaim. Berdasarkan waktu dan pola akses, situs dapat mendeteksi aktivitas yang tidak biasa jika terjadi terlalu sering.

10. Hati-hati dengan Deteksi Multi-Akun. Jika situs memiliki mekanisme deteksi multi-akun (seperti memeriksa alamat IP yang sama digunakan oleh beberapa akun), pastikan untuk menggunakan akun yang berbeda secara terpisah. Anda mungkin perlu menggunakan IP yang berbeda atau menggunakan rotasi akun secara acak.

Kesimpulan: Untuk menghindari deteksi multi-akun dan proxy dalam skrip Anda, Anda perlu memperhatikan hal-hal berikut: Rotasi Proxy dan VPN dengan IP lokal.Rotasi User-Agent dan penggunaan header yang bervariasi.Menghindari aktivitas yang terlalu cepat dan menggunakan CAPTCHA solver jika diperlukan.Gunakan proxy yang dapat dipercaya dan rotasi IP secara teratur.Namun, perlu diingat bahwa situs yang memiliki deteksi canggih mungkin masih bisa mendeteksi pola yang tidak biasa. Jika Anda terus-menerus menggunakan metode otomatis, kemungkinan akan terdeteksi seiring waktu.
